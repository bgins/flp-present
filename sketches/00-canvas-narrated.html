§ 1::the result

every protocol for this problem has the possibility of nontermination, even with only one faulty process.

[ paper §1, abstract ]
§ 2::the buffer

The message system maintains a multiset, called the message buffer, of (destination, message) pairs sent but not yet delivered.

[ paper §2 ]
▸ the buffer is one pool — not a queue per channel. The adversary can pull any pending pair in any order.
§ 2::the ∅ event

the event (p, ∅) can always be applied to C, so it is always possible for a process to take another step.

[ paper §2 ]
▸ this is the move the adversary can always make. Time passes for p without anything being delivered — the slow / dead indistinguishability in primitive form.
§ 3::valency

C is bivalent if |V| = 2.

[ paper §3 ]
— Robinson: read bivalent as "undecided yet" — the future is structurally open, even if no process has written its y register.
§ 3::lemma 3 · preservation

Let C be a bivalent configuration of P, and let e = (p, m) be an event that is applicable to C. … Then, 𝒟 contains a bivalent configuration.

[ paper §3 ]
— Aspnes: "if the adversary can keep the protocol in a bivalent configuration forever, it can prevent the processes from ever deciding."
